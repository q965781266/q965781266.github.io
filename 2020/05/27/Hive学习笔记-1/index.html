Hive学习笔记-1

Hive基本概念

Hive：由Facebook开源用于海量结构化日志的书就统计。
Hive是基于Hadoop的一个==数据仓库工具==，可以将==结构化的数据文件==映射为一张表，并提供==类似于SQL==（HQL）的查询功能。

Hive是一个==计算==引擎，本质是==将HQL转化为MapReduce程序==

说明：

  1. Hive处理的数据存储在HDFS
  2. Hive分析数据的底层的==默认实现==是MapReduce（可以进行修改）
  3. 执行程序运行在Yarn上

Hive的优缺点

优点

  1. HQL语法类似于SQL，开发人员容易上手
  2. 避免写MapReduce，减少开发人员学习成本
  3. Hive延迟比较高，在处理大数据方面有优势，常用于数据分析，但实时性要求不高
  4. 支持自定义函数， 用户可以根据自己的需求实现函数

    缺点

  5. HQL表达能力有限——迭代式算法无法表达、数据挖掘方面不擅长
  6. Hive效率低——MapReduce作业、难以调优

元数据是什么？？？

元数据（Metastore）包括：表名、表所属的数据库、表的拥有者、分区字段、表的类型（内部外部表）、表数据的目录等。
元数据默认存储在自带的derby数据库（一般存MySQL）中

Hive架构

1. SQL Parser解析器

将SQL转换成抽象语法树AST；对AST进行语法分析（表是否存在、字段是否存在、SQL语义）

2. Physical Plan编译器

将AST编译生成逻辑执行计划

3. Query Optimizer优化器

对逻辑执行计划进行优化

4. Execution执行器

将逻辑执行计划转换成MapReduce任务
